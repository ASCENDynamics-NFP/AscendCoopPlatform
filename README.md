# Ascend Co-op Platform

Ascend Co-op Platform is an open-source platform built using Ionic and Firebase, aimed at fostering collaboration and growth for worker-owned cooperatives, nonprofit organizations, and private organizations. It offers features like user profiles, group management, real-time data, and collaboration tools to address various challenges and facilitate communication between organizations.

For a deeper look at the application layout and how the frontend communicates with Firebase, see [docs/architecture.md](docs/architecture.md).


## Development

This project uses a continuous development process. As changes are made, they are immediately deployed to a development environment.

### Running Tests

To execute the unit tests for both the Angular application and Firebase functions run:

```bash
npm run test
```

This command will build the project and run all Jasmine and Mocha tests. Ensure
`@angular/cli` is installed globally and that a headless Chrome binary is
available so Karma can launch the browser.

Example installation on Ubuntu:

```bash
npm install -g @angular/cli
sudo apt-get update
sudo apt-get install -y chromium-browser
```

## Getting Started

### Prerequisites

- Node.js and npm
- Angular CLI (`npm install -g @angular/cli`)
- Firebase CLI (for deployment only — `npm install -g firebase-tools`)

1. Install Node.js, npm, and Angular CLI on your development machine.
2. Clone this repository: `git clone https://github.com/ASCENDynamics-NFP/AscendCoopPlatform.git`
3. Change to the project directory: `cd AscendCoopPlatform`
4. Install dependencies: `npm install` (Git hooks are installed automatically). If you don't see them, run `npm run prepare`.
5. Run the development server: `npm start`. The `start` script chains `generate-env:dev`, which reads the tracked `.env.development` file and writes `src/environments/environment.ts`, `src/index.html`, and `src/funding.json`.
6. Open your browser and navigate to `http://localhost:8100/` to view the app.

### Local environment variables

`.env.development` is **tracked in git** and contains the public dev Firebase config (`ascendcoopplatform-dev`) plus public OAuth client IDs, so a fresh clone runs out of the box — no secrets to fill in for local development.

`.env.production` is **gitignored**. Create it (copy from `.env.example`) only when you need to build for your own production Firebase project. The same applies to forks (see [Forking and deploying your own instance](#forking-and-deploying-your-own-instance) below).

Files generated by `generate-env.js` and therefore gitignored:

- `src/environments/environment.ts` / `environment.prod.ts` — generated from `.env.development` / `.env.production`. `src/environments/environment.example.ts` is the tracked shape reference.
- `src/index.html` — generated from `src/index.html.template`.
- `src/funding.json` — generated from `src/funding.json.template`.

If any of those files are missing, run `npm run generate-env:dev` (or `:prod`). The `npm start`, `build:dev`, and `build:prod` scripts already chain it for you.

## Forking and deploying your own instance

This codebase is designed to be forked and redeployed by other cooperatives, nonprofits, and organizations under the AGPL-3.0 license. To stand up your own branded instance:

1. **Fork & clone** the repository, then `npm install`.
2. **Provision Firebase**: create a new Firebase project, enable Auth (Email, Google), Firestore, Realtime Database, Storage, Cloud Functions, Hosting, and Remote Config.
3. **Configure env vars**: replace the upstream `.env.development` (which points at `ascendcoopplatform-dev`) with your own Firebase project, and create `.env.production` from `.env.example`. Fill in every value. New forks should pay particular attention to:
   - The Firebase web app config, the three Google OAuth client IDs, and your Google Maps key.
   - `APP_BASE_URL` — the public URL your fork will be served from (no trailing slash). Used in SEO meta tags, Open Graph, canonical link, and JSON-LD.
   - `APP_NAME`, `APP_DESCRIPTION`, `SUPPORT_EMAIL`, `PRIVACY_EMAIL` — substituted into `src/index.html`, `src/funding.json`, the legal modal, and the terms-of-use page by `generate-env.js` on each `npm start` / `npm run build:*`.
4. **Rebrand the app ID and display name** (Capacitor `appId`, Android package, iOS bundle ID, asset links, Google Sign-In configs, app name strings):

   ```bash
   # Bundle ID + display name in one call:
   npm run set-bundle-id -- com.yourorg.yourapp --name "Your Co-op Name"

   # Or just rename without changing the bundle ID:
   npm run set-bundle-id -- org.ascendynamics.coopplatform --name "Your Co-op Name"
   ```

   The `configure-app.js` script rewrites every reference (Capacitor config, `AndroidManifest.xml`, `build.gradle`, iOS `project.pbxproj`, Apple Sign-In `clientId`, assetlinks, etc.), moves the Android Java source dir to match, and (when `--name` is passed) updates `capacitor.config.ts`, `android/app/src/main/res/values/strings.xml`, `src/manifest.json`, and iOS `Info.plist` `CFBundleDisplayName`. It also syncs the Android `server_client_id` from `GOOGLE_AUTH_WEB_CLIENT_ID` in your `.env.production` (falling back to `.env.development`). It is idempotent and round-trip safe — re-run it any time to change either value again.
5. **Regenerate native credentials** that cannot be auto-patched:
   - Register the new bundle ID in the Firebase console under both iOS and Android apps.
   - Download fresh `android/app/google-services.json` and `ios/App/App/GoogleService-Info.plist`.
   - For Apple Sign-In, register the new App ID + Service ID in the Apple Developer portal.
6. **Sync native projects**: `npx cap sync`.
7. **Replace branding assets**: swap `src/assets/image/icon-512x512.png` (used as the default OG/Twitter card image) and the logos under `src/assets/image/logo/` with your own. Runtime brand tokens (logo URL, primary/secondary colors, app name) are managed live via Firebase Remote Config — see [docs/BRANDING.md](docs/BRANDING.md).
8. **Hand-edit the files outside `src/` that the rebrand script intentionally does not touch.**
   - `.firebaserc` — change both `default` and `production` to your own Firebase project IDs (or remove the production target if you only have one).
   - The informational pages under `src/app/modules/info/pages/` (about-us, landing, services, team, think-tank, etc.), the i18n strings under `src/assets/i18n/`, and the prose blocks in `src/funding.json` (plan descriptions, history) contain organization-specific copy you will likely want to rewrite for your community — but nothing there blocks a clean build.
9. **Deploy**: `npm run build:prod && firebase deploy`.

The platform reads its Apple Sign-In redirect URI from `environment.firebaseConfig.authDomain`, its public-facing URLs from `environment.appBaseUrl`, its contact emails from `environment.supportEmail` / `environment.privacyEmail`, and its display brand from Remote Config. Static-site copy in `src/index.html` and `src/funding.json` is rewritten on every `generate-env:*` run via `__TOKEN__` placeholders, so once the steps above are done no further source edits are required for a clean fork.
